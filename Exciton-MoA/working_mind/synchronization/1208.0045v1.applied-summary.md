## Paper
- Title: Synchronization in Complex Oscillator Networks and Smart Grids
- Authors: Florian Dorfler, Michael Chertkov, Francesco Bullo
- Year: 2012
- Source: https://arxiv.org/abs/1208.0045
- Local file: `1208.0045v1.pdf`

## Why this paper is in `working_mind`
This paper is the closest scholarly match to the current `Exciton-MoA` bottleneck: stabilizing communication across selectively coupled, drifting, phase-sensitive subsystems under imperfect local signals and thresholded control.

The project language uses entangled manifolds, but the engineering problem is more naturally expressed as synchronization in a nonlinear coupled network. This paper is valuable because it focuses on when synchronization is possible, how topology and coupling affect it, and how to reason about the threshold between incoherence and stable phase alignment.

## Core takeaway
Synchronization should be treated as a network feasibility and margin problem, not only as a local feedback problem.

In practice, that means the current controller should not only ask:
- should we nudge now?

It should also ask:
- is this manifold pair currently inside a synchronizable region?
- how far are we from that region?
- are we failing because of local hint weakness, or because coupling-plus-drift cannot currently support stable alignment?

## Direct relevance to current `Exciton-MoA` work
This maps cleanly onto the current stabilization effort:

- `phase_coherence`, `damping`, and control nudges already behave like a coupled oscillator control problem.
- The wormhole registry and weighted cross-manifold links are the effective network topology.
- The hint gate is acting like a conservative synchrony-admission test, but it is not yet grounded in a topology-aware synchrony margin.
- The current `observe_balanced_signal` regime may actually represent two different realities:
  - a genuinely safe basin that should stay lightly observed
  - a deceptively calm but unsynchronizable basin where small nudges will not help

## Engineering implications
1. Add a synchrony-margin concept to replay and sweep summaries.
   Instead of only reporting hint confidence and reliability, estimate a derived margin that combines cross-manifold coupling, coherence spread, drift, and local instability.

2. Separate control eligibility from synchronizability.
   A hint can be reliable yet still be acting inside a region where synchronization is structurally unlikely. That distinction is missing in the current pipeline.

3. Use topology-aware gating for future policy experiments.
   Threshold changes should ideally depend on a synchrony margin or basin indicator, not only on per-tick hint reliability.

4. Revisit the `observe` bucket.
   The current `observe_balanced_signal` label is too broad. It likely mixes stable-hold states with low-actionability states that only look balanced because the pair has already fallen into a shallow, low-variance regime.

## Best near-term use
The best immediate use of this paper is not to copy equations directly into the controller. It is to drive the next diagnostics layer:

- define a synchrony-feasibility proxy
- compare that proxy across natural-entry seeds versus near-miss seeds
- use the result to decide whether future gating should be recommendation-specific or basin-specific
